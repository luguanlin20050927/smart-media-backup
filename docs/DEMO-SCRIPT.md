# 影序 YINGXU｜三分钟演示脚本

## 0:00–0:30 先说痛点

“摄影师不是缺一个复制按钮，而是缺一个能证明素材已经安全抵达、以后还能找回的流程。”

## 0:30–1:10 插卡与识别

打开影序，插入 SD 卡。展示文件总量、照片/视频数量和设备识别结果，说明原始卡只读扫描，不会修改源文件。

## 1:10–1:50 备份与校验

选择外置盘或本地目录，开始备份。展示实时进度、复制数量、校验数量和失败提示。强调“复制完成”与“校验通过”是两件都要完成的事。

## 1:50–2:25 历史与报告

打开历史记录，进入本次任务详情，展示已复制、已校验、跳过和失败数量；下载或打开备份报告，说明每次任务都有可追溯结果。

## 2:25–3:00 增量能力

对同一来源再次执行备份，展示文件被识别为已存在并跳过，不重复写入。最后说明：影序当前优先保证本地可靠性，云同步和移动端原生能力会在后续迭代。
