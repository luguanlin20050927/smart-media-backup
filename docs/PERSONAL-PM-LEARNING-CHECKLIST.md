# 个人项目经理学习清单

## 一、下一项目开始前

- [ ] 写出目标用户和真实使用场景
- [ ] 写出用户当前的替代方案
- [ ] 用一句话说清产品价值
- [ ] 明确本次不做什么
- [ ] 建立 P0/P1/P2/P3 需求表
- [ ] 写出每个 P0 的验收条件
- [ ] 建立版本、文件和数据备份规则

## 二、产品经理能力

- [ ] 用户访谈和问题提炼
- [ ] 用户故事与使用流程
- [ ] 竞品分析和差异化定位
- [ ] 功能优先级和范围控制
- [ ] 信息架构和页面地图
- [ ] 需求变更记录
- [ ] 版本规划和里程碑管理

## 三、设计能力

- [ ] 先画低保真线框，再做高保真视觉
- [ ] 建立颜色、字体、间距、圆角、阴影规范
- [ ] 统一导航、按钮、图标和状态组件
- [ ] 设计加载、空状态、成功、失败、取消状态
- [ ] 设计手机、桌面和不同语言布局
- [ ] 每次动效都说明它服务什么信息

## 四、技术协作能力

- [ ] 能看懂项目目录和启动方式
- [ ] 能画出数据流和模块边界
- [ ] 能理解数据库关键字段
- [ ] 能读懂日志和错误堆栈
- [ ] 能提出可测试的接口条件
- [ ] 能判断哪些功能需要真实设备验证

## 五、测试与发布能力

- [ ] 正常路径测试
- [ ] 无设备测试
- [ ] 权限不足测试
- [ ] 空间不足测试
- [ ] 网络不可用测试
- [ ] 文件损坏测试
- [ ] 文件占用测试
- [ ] 中途取消和继续测试
- [ ] 目标盘断开测试
- [ ] 版本升级和旧数据迁移测试
- [ ] 官网、安装包、Release、README、SHA 统一检查

## 六、比赛与简历表达

用以下顺序讲项目：

```text
用户痛点
→ 目标用户
→ 产品方案
→ 核心创新
→ 技术实现
→ 可靠性验证
→ 实际结果
→ 后续规划
```

不要只说“我参与开发了软件”，要具体说：

> 我负责从摄影师素材备份痛点出发，完成需求拆解、产品定位、交互设计、macOS 产品实现、官网发布、异常测试和版本交付，最终形成可下载安装的影序 YINGXU。

## 七、每周复盘问题

1. 本周交付了什么可验证成果？
2. 哪个决定让项目更接近用户价值？
3. 哪个工作只是看起来很忙？
4. 哪个 Bug 本来可以更早发现？
5. 下周唯一最重要的目标是什么？
